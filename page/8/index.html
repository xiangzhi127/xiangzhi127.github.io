css属性-float

float最初的初衷是为了实现文字环绕效果。
flaot具有包裹性。
flaot具有破坏性，但它的破坏性只是为了实现环绕的效果而已。
某个元素被设置float属性后，这个元素会生成一个块级框，而不论它本身是何种元素。
(假如这个元素是行内标签，本来是不可以设置宽高的，float后，变成了块级元素，这样就可以设置它的宽高了。)
假如在一行之上只有极少的空间可供浮动元素，那么这个元素会跳至下一行，这个过程会持续到某一行拥有足够的空间为止。
浮动后浮动元素与文字处于同一文档流。
flaot具有破坏性是指：浮动后的浮动元素会使父元素的高度塌陷。那么解决这个问题的方法有
（1）父元素设置高度
（2）父元素设置overflow:auto;
JavaScript 语法： object.style.cssFloat=”left”。
清除浮动带来的影响：
1、clear:both;这个只适合于block水平元素，对于行内元素不起作用的(对于外层使用clear:both;会清除元素的跟随性，但是父元素高度塌陷还是没有解决。
所以：clear:both建议使用在内层。)。
2、伪元素清除,用于父元素上。（只适合于外层）
清除浮动的css代码：
.clearfix:after{
content:’’;
display:block;
height:0;
overflow:hidden;
clear:both;
}
.clearfix{zoom:1;}
或者
.clearfix:after{
content:’’;
display:table;
clear:both;
}
.clearfix{zoom:1;}
3、BFC/haslayout(用于外层的)
父元素设置为BFC/haslayout。
BFC/haslayout通常声明：
（1）float:left/right;
（2）position:absolute/fixed
（3）overflow:hidden/scroll/auto
（4）display:inline-block/table-cell
（5）width/height/zoom:1
3列布局：
float:left；
float:right;
text-align:center;
2列布局，一侧固定，另一侧自适应：
float:left;
margin-left;
自适应布局(固定布局写法)：
float:left;
display: table-cell; display: inline-block; width: 2000px; width: auto;
