（1）relative是相对于自身进行定位的。relative在进行定位时，如果left、right同时出现，取left值；同样如果top、bottom同时出现，取top值。
（2）根据后来居上的原则，后面的会覆盖前面的元素。设置position:absolute可以提高层级。
（3）当Position属性值为Relative时对象原来占有的位置保留，其后面的对象按原来文档流仍然保持。
（4）relative的最小化影响原则：指的是尽量减低relative属性对其他元素或布局的潜在影响

前端优化

1、优化的基本准则
按需加载（只加载你需要的）
并行（让串行的事情并行起来）
压缩（通过压缩减少体积）
缓存（利用缓存，减少请求等待）
预测（预测用户行为，提前发出请求）
合并（把多个零散文件合并起来，减少请求）
自动化（让速度优化变成一种常规，和自动化工作（例如glup ,grunt）结合，减少成本）

2、优化css

1、csslink压缩
2、使用gulp-concat将多个css合并在一起
3、不要使用@import减少阻塞和请求，可以使用CSS delivery工具检测
4、如果是移动端，建议head只放首屏能用的css,首屏外的css下移。
5、检查没有用到的css。

3、优化JavaScript

1、一个变量只存在一种类型的数据
2、尽量减少对隐式的依赖，这样可以增强程序的可读性
3、使用匈牙利命名法
4、在对当前DOM进行操作之前，尽可能多的做一些准备工作，保证N次创建，1次写入
5、判断空值用val===null即可
6、运用逻辑与代替if
7、避免全局查找，在一个函数中会用到全局对象储存为局部变量来减少全局查找，因为访问局部变量的速度要比访问全局变量的速度跟快些。

4、减少对服务器的文件请求

1、使用css sprite技术合并多个图片为单个图片文件，实际使用时通过background-position来定位背景位置。
2、合并多个css样式文件为单个样式文件，合并多个脚本为单个脚本，再在页面中引用合并后的样式/脚本文件。
3、使用base64编码来展示图片，常规上值推荐把这种方式使用在用户重复访问较少的页面，因为他们虽然无须从服务器get一遍，但也无法缓存在客户端，导致用户每次当问页面都要重新渲染页面一次。而且冗长的文件流代码会占用你页面很大的代码空间，维护其页面来估计也会挺心塞。
4、将小块的css、js代码段直接写在页面上

5、前端优化—减少文件大小

1、压缩样式/脚本文件，可以用glup或grunt来实现这点。
2、使用Font  Awesome来代替页面上的图标，其原理是使用@font-face让用户下载一个非常小的UI字体包，把页面上用到的图标以字符的形式来显示，从而减少了图片需求和图标文件大小。

6、前端优化—适度使用CDN

1、常规建议是一个站点下只使用同一个可靠、快速的CDN来引入各种所需资源即可。

7、前端优化—延迟请求、异步加载脚本

1、将脚本请求标签放置到</body>结束标签前，使得页面上的脚本为最后被请求的资源，字让不会阻塞其他页面资源的请求事件。还有就是在script标签上加上defer和async属性。
2、使用requireJS或seajs
3、<script async>将会在HTML解析时下载该文件并在下载完成后马上执行</script>
   <script defer>将会在HTML解析时下载该文件并在HTML解析完成后执行</script>

function downloadJSAtOnload() {
var element = document.createElement("script");
element.src = "defer.js";
document.body.appendChild(element);
}
if (window.addEventListener)
window.addEventListener("load", downloadJSAtOnload, false);
else if (window.attachEvent)
window.attachEvent("onload", downloadJSAtOnload);
else window.onload = downloadJSAtOnload;

11、提前获取DNS来提升网页加载速度

1、DNS预取<link rel="dns-prefech" href="//host_name_to_prefetch.com">
DNS预取是在用户尝试点击链接钱师徒解析域名。一旦域名被解析，且用户导航到该域名，则不会因DNS查找而导致加载时间变长。如果你的网页的内容来自多个域名，那么这绝对是一个聪明的、能加快页面加载速度的方法。

、前端优化—其他建议
1、不要再css中使用import，它会让一个样式文件去等待另一个样式文件的请求，无形中增加了页面等待时间。
2、避免页面或者页面文件重定向查找。
3、减少无效请求
4、无论你是否决定将脚本放到页尾，但一定要保障脚本放置于样式文件后方。
5、文件在小于50K的时候，直接读取文件流会比从文件系统中去读取文件来得快些，比如有一张图片，如果它小于50K,我们可以将它转为二进制数据存储在数据库中，页面若要读取该图片则从数据库上来读取。
6、使用cookie-free domains来存放资源，减少无用cookie传输的网络开销
7、配置htaccess文件、走Gzip页面压缩形式，并开启keep-alive连接模式等后端解决方案。

、前端优化：DNS预解析提升页面速度
浏览器对网站第一次的域名DNS解析查找流程依次为：浏览器缓存-系统缓存-路由器缓存-ISP DNS缓存-递归搜索。
查找DNS是需要花费时间的，经验总结至少需要20毫秒左右的时间，在此期间，浏览器是无法下载其他内容。所以浏览器会想办法对DNS查询进行缓存。但只会缓存一段时间。
减少DNS查找次数时，最理想的方法就是将所有内容放在同一个域。这样访问整个网站就只需要一次DNS查找，可以提高性能。但是可能会造成下载资源时出现排队的问题。所以建议一个网站里，至少2个域名，但不多于4个域。

JavaScript-语法

1、区分大小写
ECMAScript中的一切，包括变量、函数名和操作符都是区分大小写的。
2、标识符
所谓标识符，就是指变量、函数、属性的名字，或者函数的参数。标识符可以是下列格式规则组合起来的一或多个字符：第一字符必须是一个字母、下划线(_)或一个美元符号($)；其他字符可以是字母、下划线、美元符号或数字。
不能把关键字、保留字、true、false和null作为标识符。
3、注释
ECMAScript使用C风格的注释，包括单行注释和块级注释。

4、Number()函数的转换规则
（1）如果是Boolean值，true和false分别被转为1和0
（2）如果是数字值，只是简单的传入和返回
（3）如果是null值，返回0
（4）如果是undefined，返回NAN
（5）如果是字符串，且只包含数字（包括正负），则将其转换成十进制数值（如果有前导0,0会被忽略）
（6）如果是字符串，且包含有效的浮点格式，，则将其转换成对应的浮点数值。
（7）如果是字符串，且包含有效的十六进制格式，则将其转换成相同大小的十进制整数。
（8）如果是字符串，且为空，则将其转换成0
（9）如果是字符串，且不为上述的（5）（6）（7）（8），则返回NAN
5、parseInt()函数在转换字符串时，它会忽略字符串的空格，直至找到第一个非空格字符。如果第一个字符串不是数字字符或者负号，则返回NAN。
（1）如果是空字符串，返回NAN
6、parseFloat(),字符串的第一个小数点是有效的，第二个是无效的。十六进始终装换成0。
