css属性padding

对于block水平元素：
1、padding值很大，一定会影响尺寸（不管你是否设置了box-sizing:border:box;）
2、width为非auto时，padding影响尺寸。
3、当元素有width:auto或者box-sizing:border-box时，同时padding值并不是很大，元素尺寸不变。
4、这里的padding很大是指大于元素尺寸的50%的值。
对于inline元素：
影响尺寸，尤其是有背景的时候。
inline元素利用padding实现分割线：
.span{
padding: 16px 6px 1px;
border-left: 2px solid #ccc;
font-size: 0;
margin-left: 12px;
}
padding负值：
padding不支持任何负值。
padding百分百值：
padding百分比是相当于宽度计算的。
利用padding百分百实现一个正方形。
padding：50%；
标签元素内置padding：
很多标签元素本身内置了margin、padding值，它们的单位是以em为单位的，
1、ol/ul元素内置padding-left，但单位是px不是em
2、例如chrome浏览器下是40px
3、字号很小时，间距就会很开
4、字号很大，序号会爬到容器外面
所有浏览器input/textarea输入框内置padding
所有浏览器button按钮内置padding
部分浏览器select下拉内置padding，如Firefox 、IE8+可以设置padding
所有浏览器radio、checkbox单复选框无内置padding
button按钮的padding最难控制。
firefox浏览器button设置padding:0后，左右还是有padding
解决方法：button::-moz-focus-inner{padding:0;}
IE7浏览器中，button里面的文字越多，左右的padding就越大。
解决方案：button{overflow：visible}

html面试题

、web标准的理解
“WEB标准”是一系列标准的总称，包括HTML4.0、XHTML1.1、CSS2.1、XML1.0、RSS2.0、ECMAScript1.1、DOM1.0等等。可以简单表示为以下的几点：
标签闭合、标签小写、标签不乱嵌套、使用外链css和js、三层分离、文件下载和访问页面速度跟快、更少的代码。

、xhtml与html有什么区别？
最主要不同的是：
XHTML元素必须被正确嵌套
XHTML元素必须被关闭
XHTML标签名必须小写
XHTML文档必须由根元素

、什么是DOCTYPE？严格模式和混杂模式有什么区别？它们有何意义？
DOCTYPE标签是文档类型的缩写，<!doctype>用于声明一个页面的文档类型定义（DTD）。
特点：
（1）doctype标签是单独出现的，没有结束标签。
（2）文档类型定义在HTML文档的第一行，在html标签之前。
（3）文档类型，会使浏览器使用相应标准加载网页并显示。
（4）当文档不定义DOCTYPE，浏览器将无法获知HTML或XHTML文档的类型，因此会进入怪异模式。

、HTML5为什么只需要写<!DOCTYPE HTML>？
HTML5不基于SGML，因此不需要对DTD进行引用，但是需要doctype来规范浏览器的行为。
其他版本的基于SGML，所以需要对DTD进行引用，才能告知浏览器使用哪种规范来解析文档。

、简述一下你对HTML语义化的理解？
（1）可以让页面的内容结构化，便于浏览器、搜索引擎解析

、img标签上的title与alt属性的区别？
alt属性是在你的图片因为某种原因不能加载时在页面显示的提示信息，它会直接输出在原本加载图片的地方，而title属性是在你鼠标悬停在该图片上时显示一个小提示，鼠标离开就没有了

、行内元素有哪些？块级元素有哪些？空（void）元素有哪些？
行内元素：不能够设置元素的尺寸，它只能自适应内容，无法隔离其他元素。
块级元素：能够设置元素尺寸，可以隔离其他元素。
行-内块元素：可以设置尺寸，但无法隔离其他元素。
空元素：空元素表示在开始标签中关闭的
行内元素：span、em、i、strong 、a、input、img、label
块级元素：div、p、h1~h6、form、ul、li

、title与h1的区别、b与strong的区别、i与em的区别？
h1：突出文章主题
title:突出网站标题或关键字，从seo角度，title比h1权重高
b:加粗
strong：加粗，但语气上带有强调的意思
i:倾斜
em:倾斜，但语气山带有强调。

、html5有哪些新特性、移除了哪些元素？如何处理HTML5新标签的浏览器兼容问题？
新特性：
拖拽释放(Drag and drop) API
语义化更好的内容标签（header,nav,footer,aside,article,section）
音频、视频API(audio,video)
画布(Canvas) API
地理(Geolocation) API
本地离线存储 localStorage 长期存储数据，浏览器关闭后数据不丢失；
sessionStorage 的数据在浏览器关闭后自动删除
移除元素：
纯表现的元素：basefont，big，center，font, s，strike，tt，u；
标签兼容性：
IE8/IE7/IE6支持通过document.createElement方法产生的标签，可以利用这一特性让这些浏览器支持HTML5新标签。

、html5有哪些新的API？
拖拽释放(Drag and drop) API
画布(Canvas) API
地理(Geolocation) API

、HTML5的离线存储怎么使用，工作原理是怎样的？
HTML5的离线存储是基于一个新建的.appcache文件的，通过这个文件上的解析清单离线存储资源，这些资源就会像cookie一样被存储了下来。之后当网络在处于离线状态下时，浏览器会通过被离线存储的数据进行页面展示。

、HTML5的form如何关闭自动完成功能？
autocomplete=”off”

、iframe有那些优缺点？
优点：
（1）可以增加代码的可重用
（2）如果有多个网页引用iframe，那么你只需要修改iframe的内容，就可以实现调用的每一个页面内容的更改，方便快捷
缺点：
（1）iframe会阻塞主页面的onload事件
（2）很多的移动设备（PDA 手机）无法完全显示框架，设备兼容性差
（3）iframe框架页面会增加服务器的http请求，对于大型网站是不可取的
