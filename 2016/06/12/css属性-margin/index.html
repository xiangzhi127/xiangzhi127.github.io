css属性-margin

有什么条件导致margin是无效的？？
（1）

margin的特性1：margin可以改变元素的尺寸,但必须满足一些前提条件：
（1）使用于没有设定width/height的普通block水平元素。（float、absolute、fixed都可以让元素变成block，但不是普通元素了）
（2）只适合用于水平方向尺寸（即只可以改变水平方向的长度）
应用：滚动容器内上下留白。
margin设置为百分比后：
普通元素的百分比margin都是相对于容器的宽度计算的。
绝对定位元素的百分比margin是相对于第一个定位祖先元素（relative/absolute/fixed）的宽度计算的
应用：高2:1自适应矩形
父：{
overflow：hidden；
}
子：{
margin:50%;
}
margin重叠：
margin重叠通常特性：
1、block水平元素（不包括float和absolute元素）
2、不考虑writing-mode,值发生在垂直方向（margin-top/margin-bottom）
margin重叠3种情景：
1、相邻的兄弟元素
2、父级和第一个/最后一个子元素
3、空的block元素
margin-top重叠条件（指第2中情景）：
1、父元素非块状格式化上下文元素
2、父元素没有border-top设置
3、父元素没有padding-top值
4、父元素和第一个子元素之间没有inline元素分割
干掉margin-top：
1、父：{overflow：hidden}
2、父：{border:1px solid #ccc}
3、父：{padding:10px}

margin-bottom重叠条件（指第2种情景）：
1、父元素非块状格式化上下文元素
2、父元素没有border-top设置
3、父元素没有padding-top值
4、父元素和第一个子元素之间没有inline元素分割
5、父元素没有height，min-height，max-height限制。
干掉margin-bottom：
1、父：{overflow：hidden}
2、父：{border:1px solid #ccc}
3、父：{padding:10px}
4、之间插入inline元素：比如 
5、加个height
空block元素重叠条件：
1、元素没有border设置
2、元素没有padding值
3、里面没有inline元素
4、没有height、或者min-height
干掉空block元素重叠：按照上面的规则来就行。
重点知识，margin:auto
absolute与margin实现垂直居中
父：{height:200px;position:relative;}
子：{position:absolute;top:0;right:0;bottom:0;left:0;
width:500px;height:100px;margin:0 auto;}
margin负值定位：
1、margin负值下的两端对齐
margin-start:
正常的流向，margin-start等同于margin-left,两者重叠不累加。
如果水平流是从右往左，margin-start等同于margin-right。
在垂直留下（writing-mode:vertical-*;）,margin-start等同于margin-top。
